## 1. 저장공간 / 캐시 정책 검증

- EC2 디스크 20GB 중 12GB 사용, 8.1GB 여유 (kr-scalping이 대부분)
- BBC 프로젝트 총 사용량: 544 KB (3일치) — 일 평균 ~172 KB
- 10년 운용 예측: <1 GB → **정리 cron 불필요**, 그대로 누적
- 번역 캐시는 URL 해시 키로 디스크 공유 → **모든 세션이 같은 캐시 사용**, gemini 호출은 기사당 평생 1회

| ⭐ 즐겨찾기 | 카드 버튼 | 라디오 `⭐ 저장됨` 탭에서 모든 날짜 누적 표시 |

### 아키텍처 변경
- `fetch.py` 구조 개편:
  - `CATEGORIES`: 카테고리 메타 (group, label) — 화면 표시
  - `FEEDS`: 실제 RSS URL 리스트 — 한 카테고리에 다중 source 가능
- `Article` 데이터클래스에 `source` 필드 추가
- Streamlit 카드에 출처 배지 표시 (`TechCrunch` `Guardian` 등)
- 출처 캡션은 오늘 실제 등장한 source만 동적 나열

### 피드 검증 결과
21개 피드 (BBC 16 + 외부 5) 모두 정상 작동, 회당 최대 126건 (--per 6).

## 4. 본문 추출 다중 매체 대응 (trafilatura)

- 기존 `scrape.py`는 BBC HTML 구조 (`data-component="text-block"`) 전용
- TechCrunch/Guardian/Verge 등에서 "본문 영역을 찾지 못함" 발생
- **`trafilatura` 도입** — 뉴스 사이트 본문 추출 표준 라이브러리
- 2단계 폴백: trafilatura → BeautifulSoup 휴리스틱 (article → main → all `<p>`)
- 실패 시 캐시 미저장 → 다음 클릭 때 새 코드로 자동 재시도
